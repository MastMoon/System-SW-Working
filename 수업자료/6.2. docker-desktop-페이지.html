Docker Desktop 사용 가이드

2025년 4월 9일

1. Docker Desktop 소개

  • Docker Desktop은 Mac, Windows, Linux를 위한 Docker 컨테이너 관리 애플리케이션
  • 컨테이너를 손쉽게 관리할 수 있는 GUI 환경 제공
  • Docker Engine, Docker CLI, Docker Compose, Kubernetes 등 포함
  • 로컬 개발 환경에서 컨테이너를 쉽게 관리할 수 있는 도구
3

2. Docker Desktop 설치

Windows 설치

  • Windows 10 64비트: Pro, Enterprise, Education (Build 18362 이상)
  • Windows 10 Home (Build 19041 이상)
  • WSL 2 기능 활성화 필요
  • 도커 웹사이트에서 설치 파일 다운로드

Mac 설치

  • Intel 프로세서 또는 Apple Silicon 맥 지원
  • macOS Monterey 이상 권장
  • 도커 웹사이트에서 설치 파일 다운로드
4

3. Docker Desktop 인터페이스

  • Containers: 실행 중인 컨테이너 목록 및 관리
  • Images: 다운로드된 이미지 관리
  • Volumes: 데이터 볼륨 관리
  • Dev Environments: 개발 환경 관리
  • Extensions: 확장 기능 관리
5

4. 컨테이너 실행하기

GUI로 컨테이너 실행

  1. Docker Desktop 실행
  2. 'Images' 탭 선택
  3. 'Run' 버튼 클릭
  4. 컨테이너 설정 (포트, 볼륨, 환경변수 등) 지정
  5. 'Run' 버튼 클릭하여 컨테이너 시작

명령어로 컨테이너 실행

# 기본 실행
docker run -d -p 80:80 nginx

# 이름 지정 및 볼륨 마운트
docker run -d --name my-nginx -p 8080:80 -v ./html:/usr/share/nginx/html nginx
6

5. 컨테이너 관리하기

컨테이너 목록 확인

  • 'Containers' 탭에서 실행 중인 모든 컨테이너 확인
  • 상태, 포트, 이름 등 정보 확인 가능

컨테이너 조작

  • 시작/중지: 컨테이너 이름 옆 재생/정지 버튼
  • 삭제: 컨테이너 이름 옆 휴지통 아이콘
  • 로그 확인: 컨테이너 이름 클릭 후 'Logs' 탭
  • 터미널 접속: 컨테이너 이름 클릭 후 'Exec' 탭
7

6. 이미지 관리하기

이미지 검색 및 다운로드

  1. 'Images' 탭 선택
  2. 검색창에 이미지 이름 입력
  3. Docker Hub에서 검색된 이미지 목록 확인
  4. 'Pull' 버튼 클릭하여 다운로드

이미지 관리

  • 삭제: 이미지 이름 옆 휴지통 아이콘
  • 태그 관리: 이미지 이름 클릭 후 태그 관리
  • 이미지 내보내기/가져오기: 'Save' 및 'Load' 버튼
8

7. Docker Compose 사용하기

Compose 파일 생성

version: '3'
services:
  web:
    image: nginx
    ports:
      - "8080:80"
  db:
    image: mysql:5.7
    environment:
      MYSQL_ROOT_PASSWORD: example

Docker Desktop에서 Compose 실행

  1. 'Containers' 탭에서 'Compose' 버튼 클릭
  2. docker-compose.yml 파일 위치 지정
  3. 'Start' 버튼 클릭하여 컴포즈 실행
참고: Docker Desktop에서는 여러 서비스로 구성된 애플리케이션을 쉽게 관리할 수 있습니다.
9

8. 볼륨 관리하기

볼륨 생성

  1. 'Volumes' 탭 선택
  2. 'Create' 버튼 클릭
  3. 볼륨 이름 입력 후 생성

볼륨 사용

  • 컨테이너 실행 시 'Volumes' 설정에서 볼륨 지정
  • 볼륨 데이터 확인: 볼륨 이름 클릭 후 'Data' 탭
  • 볼륨 삭제: 볼륨 이름 옆 휴지통 아이콘
10

9. Docker Desktop 설정

리소스 설정

  • CPU: 할당할 CPU 코어 수 조정
  • 메모리: 할당할 메모리 양 조정
  • 디스크: 이미지 저장 위치 및 디스크 크기 조정

네트워크 설정

  • 프록시 설정
  • Docker 네트워크 구성
  • 포트 매핑 관리
11

10. 문제 해결

로그 확인

  • 'Troubleshoot' 메뉴에서 로그 확인 가능
  • Docker Desktop 로그 파일 위치:
    • Windows: %APPDATA%\Docker\log.txt
    • Mac: ~/Library/Containers/com.docker.docker/Data/log/vm/

일반적인 문제 해결

  • Docker Desktop 재시작
  • WSL 2 백엔드 재설정 (Windows)
  • 디스크 이미지 리셋
팁: 문제가 지속된다면 Docker Desktop을 제거하고 다시 설치하는 것도 좋은 방법입니다.
